How it works (short):
The app converts the secret text to bytes, stores its length (32-bit) followed by the message bits, and writes one bit into each color channel's least-significant bit (R,G,B) for each pixel. That provides ~3 bits per pixel capacity.
Limitations: This is a demo. LSB can be fragile (image recompression may ruin hidden text). For robust steganography use advanced transforms and consider encryption.
